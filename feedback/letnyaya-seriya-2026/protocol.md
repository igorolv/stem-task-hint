# Протокол обкатки — «Летняя серия 2026»

## Идея

Карточки подсказок генерируются по задачам, которые **уже решены**.
Решивший знает решение, поэтому может оценить подсказку по всем осям
(верность направления, спойлерность, полезность), не испортив себе задачу.

## Шаги

1. Выбираем 5–8 решённых задач, желательно разных типов
   (геометрия / теория чисел / комбинаторика / алгебра).
2. Для каждой генерируем карточку скиллом (card mode) → `cards/NN.md`.
3. Карточка читается **сверху вниз, по одному уровню**. После каждого
   уровня — ответы на вопросы формы (ниже).
4. Ответы записываем в `results.md`.

## Форма оценки — на каждый уровень карточки

| Вопрос | Варианты |
|---|---|
| Направление верное? | верное / неверное / непонятно |
| Спойлерность | 0 — не выдаёт ничего лишнего; 1 — чуть-чуть; 2 — выдаёт метод; 3 — выдаёт ключевую идею |
| Помогла бы, если бы я застряла? | да / наверное / нет |
| Комментарий | свободный текст |

## Форма оценки — на карточку целиком

- На каком уровне ты бы остановилась, если бы реально застряла на этой задаче?
- Чего в карточке не хватило? Что лишнее?
- Тон нормальный? (не поучающий, не заискивающий)

## results.md — формат

```markdown
| Задача | Уровень | Направление | Спойлерность | Помогла бы | Комментарий |
|---|---|---|---|---|---|
| 3 | 1 | верное | 1 | да | ... |
| 3 | 2 | верное | 2 | наверное | ... |
```

## Что чиним по результатам

Каждая ось фидбека указывает на свой раздел скилла:

- **Неверное направление** → протокол «реши сам → проверь → подсказывай»
  (Step 2 в SKILL.md): усилить требование верификации или честности.
- **Высокая спойлерность на ранних уровнях** → калибровка лестницы
  (раздел "The hint ladder"): правила оценки информационной цены.
- **«Не помогла бы»** → типы подсказок и их формулировки: возможно, нужен
  другой тип толчка (например, вопрос вместо факта).
- **Тон** → раздел "Tone".
