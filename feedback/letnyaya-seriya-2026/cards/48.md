# Подсказки — задача 48 (Летняя серия 2026) · v1

Числа a, b, c и d таковы, что a² + b² + c² + d² = 4. Докажите, что

(2 + a)(2 + b) ⩾ cd

**Примечание к условию:** в печатном листке неравенство строгое («>»). В такой
форме утверждение неверно — доказывается нестрогая форма «⩾»; к ней и ведёт
карточка.

**Домен:** математика (алгебра, неравенства) | **Уверенность:** проверено

> В карточке нет решения. Читай по одному уровню и останавливайся, как
> только появилась своя идея.

## Уровень 1 — ориентир

Раз неравенство нестрогое (см. примечание) — равенство где-то достигается.
Найди хотя бы один случай равенства: простые значения подбираются быстро.
Точка равенства — ориентир, по которому выверяется всё доказательство.

## Уровень 2 — область / метод

В доказываемом неравенстве c и d участвуют только через произведение cd, а в
условии — только как c² + d². Оцени произведение через сумму квадратов и
подставь условие: останется неравенство про одни a и b.

## Уровень 3 — структура

Оставшееся неравенство по a и b — после умножения на два — собирается в один
полный квадрат.

---

**Приватный разбор (в карточку не идёт, здесь — для истории).**

*Опечатка в листке.* В PDF напечатано строгое «(2 + a)(2 + b) > cd» (проверено
кодпоинтом U+003E и зум-рендером; «⩾» в этом PDF извлекается корректно — задача
34). Строгая форма опровергается: a = b = −1, c = d = 1 (условие 4 ✓, обе части
равны 1); ещё a = 0, b = −2, c = d = 0 (0 = 0). Верна нестрогая форма. По
Honesty rules карточка не может молча вести к строгому (недоказуемо) и не может
молча подменить знак (оценщица сверит с листком — кризис доверия, урок итерации
1) — поэтому явное примечание. Его неустранимая цена: сообщает «равенство
достижимо» (это следует из самих слов «строгая форма неверна при верной
нестрогой»); точки равенства примечание не называет.

*Решение.* При a² + b² + c² + d² = 4 верно тождество
2·[(2+a)(2+b) − cd] = (a+b+2)² + (c−d)² ⩾ 0
(безусловная форма: 2·[(2+a)(2+b) − cd] = (a+b+2)² + (c−d)² + (4 − a²−b²−c²−d²)).
Равенство ⟺ a + b = −2 и c = d (с условием — семейство: a ∈ [−2, 0], b = −2 − a,
c = d = ±√((4−a²−b²)/2)). Дискавери-путь тот же в два шага: 2cd ⩽ c² + d² =
4 − a² − b² (из условия), после чего остаётся 2·(2+a)(2+b) − (4−a²−b²) =
(a+b+2)² ⩾ 0.

*Проверка.* Тождество — численно на 1M случайных точек без ограничения
(расхождение ~1e−13, float-шум); минимум разности на сфере — 2M случайных точек,
→ 0⁺; семейство равенства даёт тождественный ноль. Уверенность: проверено.
См. ../results.md, итерацию по задаче 48.

*Крукс — один:* исключить c и d, связав cd (входит в доказываемое) с c² + d²
(входит в условие) стандартной оценкой (c−d)² ⩾ 0; оставшееся по a, b собирается
в полный квадрат (a+b+2)². Это одна идея — SOS из двух квадратов; «подстановка
4 = Σ квадратов сразу во всё неравенство» — та же выкладка в другой записи, не
отдельный путь. Аналитические пути (множители Лагранжа, сглаживание) — вне
школьного стиля и с бо́льшим числом случаев.

*Лестница из 3 уровней (ориентир → метод → структура).* Ур.1 — метаход нового
для обкатки типа: «найди случай равенства» — дешёвый ориентир, ОТДЕЛЬНЫЙ от
метода (отвечается подбором простых значений из самого условия: a=b=c=d=−1
находится мгновенно; не называет ни оценку произведения, ни SOS). Он опирается
на примечание к условию (равенство достижимо) — цена этой информации уже
уплачена честностью примечания, ур.1 добавляет только сам метаход. Ур.2 — метод:
асимметрия «cd в неравенстве ↔ c²+d² в условии» + оценка произведения суммой
квадратов (2cd ⩽ c²+d² не расшифровываю — базовый факт, разжёвывать = поучение).
Ур.3 — структурный факт: остаток по a, b — полный квадрат (второй квадрат SOS;
первый, (c−d)², зашит в оценку ур.2). Несёт новый концептуальный шаг — само
узнавание (a+b+2)² остаётся за учеником. Ступень-аналог не включена: после
метода остаётся вычисление + одно узнавание квадрата, аналог его лишь
пересказал бы (урок задачи 50). Именованной теоремы нет — классического ур.4
нет. Ожидаемая монотонность: ур.1 низкий → ур.2 средний → ур.3 высокий.

*Множественные пути / fallback — НЕ применяются.* Путь сущностно один (см.
крукс); fallback-секции нет, пометки «путь не единственный» нет — осознанный
no-op механизма множественных решений, третий подряд (задачи 35, 26, 48).

*Фокусы этой итерации (для проверки дочкой): (1) первый случай, когда протокол
«реши → проверь» поймал опечатку в самом условии — работает ли честное
примечание (доверие не подорвано? не спойлерит лишнего?); (2) новый тип
ориентира ур.1 — метаход «найди случай равенства»; (3) механизм путей — no-op
×3.*
